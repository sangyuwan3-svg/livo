# MID-360 建图与重定位流程

## 1. 接线与上电

1. 给 MID-360 上电。
2. 用网线连接电脑和雷达。
3. 确认电脑有线网口名称：

```bash
ip -br link
```

当前工程默认使用 `eno1`。

## 2. 配置电脑网口 IP

```bash
cd ~/lidar
./scripts/setup_mid360_network.sh eno1 192.168.1.50/24
```

检查网口状态：

```bash
ip -4 addr show eno1
```

正常应能看到：

```text
192.168.1.50/24
```

## 3. 配置雷达 IP

当前已知雷达 IP 是：

```text
192.168.1.124
```

写入工程配置：

```bash
cd ~/lidar
./scripts/configure_mid360.py --host-ip 192.168.1.50 --lidar-ip 192.168.1.124
```

检查雷达是否连通：

```bash
ping 192.168.1.124
```

如果 ping 不通，先检查雷达供电、网线、网口是否是 `eno1`。

## 4. 编译工程

```bash
cd ~/lidar
./scripts/build_lidar.sh
```

以后只改本工程包时，可以用：

```bash
cd ~/lidar
source /opt/ros/humble/setup.bash
colcon build --symlink-install
```

## 5. 建图前准备

建图前建议备份旧地图：

```bash
cd ~/lidar
mkdir -p maps
cp maps/mid360_map.pcd maps/mid360_map_backup_$(date +%Y%m%d_%H%M%S).pcd
```

如果旧地图不存在，`cp` 报错可以忽略。

## 6. 启动建图

```bash
cd ~/lidar
./scripts/run_mid360_fast_lio.sh fast_lio_rviz:=true
```

启动后先让车静止 10 秒，再开始移动。

建图时注意：

1. 起点附近多停几秒。
2. 慢速移动。
3. 少急转、少原地快速旋转。
4. 尽量让雷达看到墙面、柱子、桌腿、门框等稳定结构。
5. 少对着大面积玻璃、纯白墙、空旷区域。
6. 回到起点附近后再停几秒。

## 7. 检查建图状态

另开一个终端：

```bash
cd ~/lidar
source scripts/source_lidar.sh
```

检查雷达点云：

```bash
ros2 topic hz /livox/lidar
```

检查雷达 IMU：

```bash
ros2 topic hz /livox/imu
```

检查 FAST-LIO 里程计：

```bash
ros2 topic hz /Odometry
```

检查注册点云：

```bash
ros2 topic hz /cloud_registered
```

检查车体旋转中心坐标：

```bash
ros2 topic echo /lidar/point
```

普通建图模式下，`/lidar/point` 的 `frame_id` 通常是：

```text
lidar_start
```

这表示当前输出是本次启动后的相对坐标。

## 8. 保存地图

建图完成后，不要关闭建图进程，另开终端执行：

```bash
cd ~/lidar
./scripts/save_fast_lio_map.sh
```

地图保存路径：

```bash
~/lidar/maps/mid360_map.pcd
```

确认地图存在：

```bash
ls -lh ~/lidar/maps
```

应看到类似：

```text
mid360_map.pcd
```

## 9. 关闭建图

保存地图后，回到建图终端，按：

```text
Ctrl+C
```

关闭建图、驱动和 RViz。

## 10. 启动重定位

确保雷达仍然连通：

```bash
ping 192.168.1.124
```

启动重定位：

```bash
cd ~/lidar
./scripts/run_mid360_relocalization.sh
```

该命令会加载：

```bash
~/lidar/maps/mid360_map.pcd
```

RViz 中会显示已保存地图 `/localization/map`。

## 11. 检查重定位状态

另开终端：

```bash
cd ~/lidar
source scripts/source_lidar.sh
```

检查地图是否发布：

```bash
ros2 topic echo --once /localization/map
```

检查重定位输出：

```bash
ros2 topic hz /map_lidar/odom
```

检查公开坐标输出：

```bash
ros2 topic echo /lidar/point
```

重定位成功时，`/lidar/point` 的 `frame_id` 应该是：

```text
map
```

这时 `/lidar/point` 是车体旋转中心 `base_link` 在地图坐标系下的位置。

## 12. 查看坐标与姿态

查看当前车体旋转中心坐标：

```bash
cd ~/lidar
source scripts/source_lidar.sh
ros2 topic echo /lidar/point
```

只看一条坐标：

```bash
cd ~/lidar
source scripts/source_lidar.sh
ros2 topic echo --once /lidar/point
```

输出格式类似：

```text
header:
  frame_id: map
point:
  x: 0.123
  y: -0.045
  z: 0.002
```

含义：

- `x`：车体旋转中心在当前坐标系下的 x 坐标，单位是米
- `y`：车体旋转中心在当前坐标系下的 y 坐标，单位是米
- `z`：车体旋转中心在当前坐标系下的 z 坐标，单位是米
- `frame_id`：这组坐标所属坐标系

判断当前坐标模式：

- `frame_id: lidar_start`：普通建图/里程计模式，坐标是本次启动后的相对坐标
- `frame_id: map`：重定位模式，坐标是保存地图坐标系下的全局坐标

查看完整位姿，包括位置和姿态四元数：

```bash
cd ~/lidar
source scripts/source_lidar.sh
ros2 topic echo /lidar/pose
```

使用脚本打印更易读的位姿：

```bash
cd ~/lidar
./scripts/print_lidar_pose.sh
```

只查看 `x`、`y` 和车头朝向 `yaw`：

```bash
cd ~/lidar
./scripts/print_xy_yaw.sh
```

只看一条 `x/y/yaw`：

```bash
cd ~/lidar
./scripts/print_xy_yaw.sh --ros-args -p messages:=1
```

输出格式类似：

```text
frame=map x=0.123 y=-0.045 yaw=-0.02deg
```

## 13. 验证车体中心 TF

当前外参配置：

```text
base_link -> livox_frame:
  x = -0.2547 m
  y = -0.23827579454 m
  z = 0.0 m
  yaw = +90 deg
```

这里的 `base_link` 保持机械车体坐标系：

- `base_link +X` 指向机械车体 `+X`
- `base_link +Y` 指向机械车体 `+Y`
- `./scripts/print_xy_yaw.sh` 只对显示出来的 `yaw` 做 `-90deg` 修正
- 当车头沿机械车体 `+Y` 时，脚本输出的 `yaw` 应接近 `0deg`

查看 TF：

```bash
source ~/lidar/scripts/source_lidar.sh
ros2 run tf2_ros tf2_echo base_link livox_frame
```

## 14. 如果重定位不准

先确认是否在地图坐标系：

```bash
ros2 topic echo --once /lidar/point
```

如果 `frame_id` 是：

```text
map
```

说明重定位正在输出地图坐标。

回到建图起点附近后，先停 2-3 秒再读坐标。

如果偏差仍然较大：

1. 重新建图，起点静止 10 秒。
2. 建图过程中慢速移动。
3. 回到起点附近多停几秒。
4. 确认雷达安装牢固。
5. 确认车体中心外参是否准确。
6. 确认启动重定位的位置和地图有足够重合。

## 15. 如果重定位没有输出

检查节点：

```bash
ros2 node list
```

应包含：

```text
/livox_lidar_publisher
/laser_mapping
/map_localizer
/lidar_pose_from_odom
```

检查输入 topic：

```bash
ros2 topic hz /livox/lidar
ros2 topic hz /livox/imu
ros2 topic hz /Odometry
ros2 topic hz /cloud_registered
```

检查地图文件：

```bash
ls -lh ~/lidar/maps/mid360_map.pcd
```

如果 `/map_lidar/odom` 没有数据，说明地图匹配还没有成功。

## 16. 手动给初始位姿

如果机器人不是从建图起点附近启动，在 RViz 中点击：

```text
2D Pose Estimate
```

然后在地图上给一个大概位置和朝向。

给完后检查：

```bash
ros2 topic hz /map_lidar/odom
ros2 topic echo /lidar/point
```

## 17. 停止重定位

在运行重定位的终端按：

```text
Ctrl+C
```
